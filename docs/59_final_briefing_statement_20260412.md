## 核心结论二：识别-推断-干预三层都有证据支撑，但链路断层

| 维度 | 说明 | 证据来源 |
|------|------|--------|
| **主张** | "识别神经非典特征" → "推断短期状态与长期画像" → "分层干预"的完整链路在医疗/康复端已有相对成熟证据，但迁移到教育场景时缺少等价质量的长期跟踪与多群体 RCT。这构成了你们系统设计的直接任务清单。 | — |
| **识别层** | 女性 ADHD 识别偏差与性别转诊偏差已成文献共识，包括生命周期分层与临床识别标准。ADHD 与自闭症在教育和临床场景的识别关键变量已定义（认知负荷、执行功能、感觉敏感性）。 | [证据表 B1] R2NA026 |
| **推断层** | 诊断影像学、行为生物标志物（如眼动、EEG）与认知任务表现的关联已有基线，但教学日志+行为信号推断学习者状态的实时系统稀缺。 | [证据表 B2] R2NA029 |
| **干预层** | VR/AR 沉浸式支持在 ASD 样本中有量化效果（Hedges' g=0.74；定制化 DLS 达 g=1.15）；传统适配干预（内容分块、反馈延迟、节奏调整）在高教中证据相对丰富但对神经非典型针对性设计不足。 | [证据表 B3] R2NA033<br/> [证据表 A2] R2EA023 |
| **风险提示** | 医疗端的效果量可能无法直接迁移到教学系统（干预目标、生态环境完全不同）。高教适配证据可能不适用 K-12。 | — |
| **反驳预案** | 明确区分三个迁移风险与对策：(1) **目标域转换**：医疗→学习，需重新定义评估指标（从"症状减轻"改为"学习收益+包容性体验"）；(2) **生态环境转换**：临床受控→课堂开放，需扩大样本与纵向追踪；(3) **伦理门槛**：增加隐私保护与算法透明度要求。但基础链路本身已验证，只需系统级落地工程。 | [证据表 B4] R2NA035<br/> [证据表 C1] R2NE037 |

## 核心结论三：高等教育是首发场景，但跨学段迁移证据稀缺

| 维度 | 说明 | 证据来源 |
|------|------|--------|
| **主张** | 在 AI 教育应用的快速增长中，高等教育因学生自主性高、技术接受度强、数据碎片丰富，成为首发试验场（2021-2022 年发文量翻 2-3 倍）。但向 K-12 与职业教育的迁移面临学科差异、认知发展阶段差异与机构支持差异，目前缺少成熟的迁移框架与案例。 | — |
| **高教样本优势** | PRISMA 系统综述显示，被研究人群中本科生占比 72%，绝大多数研究在高教机构；跨学段比例（涵盖中小学或成人教育的研究）<15%。 | [证据表 A2] R2EA023 |
| **跨学段尝试** | 神经非典型教育包容设计的新文献多来于高等教育（大学残障支持中心、讲座制课程调适），向中学与小学的直接翻译案例有限。 | [证据表 C2] R2NE038<br/> [证据表 C4] R2NE015 |
| **职业教育** | 40 篇文献中涉及职业/成人再学习的<3 篇，职业资格认证的适配设计几乎为零。 | — |
| **风险提示** | 可能被追问"你们是不是只能做高教，K-12 不现实"。 | — |
| **反驳预案** | (1) **阶段性策略**：先在高教中做原型验证（快速迭代），同步启动 K-12 的可行性研究（小规模试点）；(2) **差异化设计**：明确为 K-12 调整的参数包括：认知监测颗粒度（从任务级升为课堂级）、反馈节奏（从个例改为分组），伦理门槛（增加家长/监护人同意）；(3) **数据证据**：虽然跨学段 RCT 缺乏，但针对儿童与青少年神经认知发展的教学适配原则本身在发展心理学中成熟（可倒推设计）。 | [证据证明需补充儿童神经发展相关文献] |

## 核心结论四：公平性与可解释性已从"伦理宣言"转为工程约束

| 维度 | 说明 | 证据来源 |
|------|------|--------|
| **主张** | 早期 AI 教育文献强调"可能的风险"（偏见、隐私、透明度），新一波综述（2022-2026）给出可操作的工程指标：外认知负荷量化、算法可解释性需求、差异化公平性评估。这意味着你们的系统必须从原型阶段就嵌入这些约束，而不能事后补救。 | — |
| **可解释性需求** | LLM 教育应用综述总结 9 大任务类别中，至少 5 项明确提出"可解释性与人类监督需求"。PRISMA 二级证据综述呼吁更高的方法论严谨性与报告标准。 | [证据表 A3] R2EA039<br/> [证据表 A4] R2EA020 |
| **认知负荷工程化** | 神经非典型教育文献开始把"认知超负荷"量化为可测指标（内在负荷、外在负荷、相关负荷的三分法），为自适应策略的触发提供定量基础。 | [证据表 C3] R2NE035 |
| **差异化公平性** | 从"一对一"的个性化到"群体级公平"的转变：不仅问"这个学生学得好吗"，还要问"同等条件下特定群体的收益差异是否缩小"。 | [证据表 B1] R2NA026<br/> [证据表 C1] R2NE037 |
| **隐私与合规** | GDPR 与教育数据保护法规的收紧正在改变学术评估标准，部分 ML 研究因隐私承诺不足而难以复现。 | [证据表 A3] R2EA039 |
| **风险提示** | 强调这些约束可能被当成"技术门槛过高"而质疑可行性。 | — |
| **反驳预案** | (1) **设计优先**：这些约束不是"验证阶段才考虑"，而是原型设计的必选项。好消息是已有工程化工具路线（认知负荷测量、算法透明度库）；(2) **成本与收益**：满足这些约束会延长初始开发，但大幅降低部署风险与伦理审查周期；(3) **示范作用**：率先做到这些约束的系统有更强的市场与政策优势。 | [证据表 B4] R2NA035<br/> [证据表 C2] R2NE038 |

## 核心结论五：40 篇摘要覆盖足够，但全文参数补全是下一个临界任务

| 维度 | 说明 | 证据来源 |
|------|------|--------|
| **主张** | 当前 40 篇文献的摘要实读已覆盖 40/40，为框架思考与需求论证提供了充分信息。但当涉及"我们系统该用多大的学习率""反馈延迟该设多少"等工程参数问题时，需要从全文补齐：样本量与人口统计特征、统计检验方式（参数/非参），以及针对神经非典型的分层分析。 | — |
| **摘要覆盖现状** | 40/40 论文已从 Semantic Scholar、Crossref、发布商网站、开放仓储、本地保存页等来源读取摘要与关键词；已形成 CSV 索引（docs/54_crossref_abstracts_40.csv），包含主题、方法、样本大小梗概（来自摘要）。 | [证据表汇总] docs/54 |
| **全文参数缺口** | 12 篇重点文献中，已补齐全文锚点与四字段（方法、样本、指标、边界），涉及 section/DOI/paragraph ID 级定位。但剩余 28 篇属"支撑性文献"，仅有摘要级参数。 | [证据表深读] docs/55 |
| **参数补全优先级** | 优先补全的 5 类参数：(1) 样本量与类型（n、性别比、年龄分布、诊断分类）；(2) 干预频率/强度（如每周几次、单次时长）；(3) 评估时窗口（立即、1 周、8 周、12 周）；(4) 效果量（Cohen's d、Hedges' g、相对风险）；(5) 分群分析（对神经非典型的针对性结果）。 | — |
| **可循环方式** | 这个任务不需一次完成。可以随着系统设计深化而逐期补全。第 1 期先补 12 篇重点（已完成），第 2 期补 28 篇支撑性（按优先级梯次进行）。 | — |
| **风险提示** | 可能被追问"参数完全度"，或"为什么某些细节还没补"。 | — |
| **反驳预案** | (1) **口径明确**：摘要级覆盖 40/40、全文参数补全 12/40（重点）。这两个维度不混淆；(2) **渐进式补全**：按"系统设计需要什么参数"来驱动补全顺序，而不是盲目全文阅读。这样效率更高，复用性也强；(3) **备份方式**：缺少的参数可通过作者联系或预注册数据库（OSF、Zenodo）补齐，优先用已发布的补充材料而非自行猜测。 | [证据表] docs/55<br/> [证据表] docs/58 |

## 使用指南

### 组会发言「脚本」（按顺序讲述）

```
开场（30秒）：
"前面一个多月，我用40篇文献做了系统实读，用三个subagent并行工作，把这些文献从'堆列表'转成了'可执行任务清单'。我发现了5个关键洞察，直接影响我们下一步的系统设计。"

结论一（1分钟）：
"第一个发现：这个领域框架密集，验证稀缺。[展示：138篇PRISMA、五类应用场景] 
这不是说'没有理论'，而是说'框架多、落地快、长期RCT弱'。
对我们的直接启发：可以大胆借用现有框架快速迭代原型，但需要从第一天起就规划长期验证方案。"

结论二（1.5分钟）：
"第二个发现：识别-推断-干预三层都有证据，但教育场景的链路还要自己接。
[展示：医疗端的VR效果g=0.74，女性ADHD识别偏差] 
这意味着我们不是从零开始，而是'借枝接果'：用医疗/康复的识别与推断技术，配上教育自适应的干预方法。但这个嫁接需要明确的边界测试。"

结论三（1分钟）：
"第三个发现：高教是首发场景。[展示：本科样本占72%，发文增长2-3倍]
这给我们一个实际的落地路径：先在高教验证，同步做K-12的可行性研究，而不是一开始就要通吃所有学段。"

结论四（1分钟）：
"第四个发现：公平性和可解释性已经从'风险清单'变成'工程约束'。
这意味着我们不能等到验证阶段再考虑这些问题，必须在原型设计里就嵌入这些指标。好消息是工程工具已经有。"

结论五（30秒）：
"最后一个发现：40篇摘要覆盖已经完整，全文参数我已经补了12篇重点，剩余的可以按照系统需要分阶段补全。这不是'缺数据'，而是'按需补全'。"

收尾（30秒）：
"综合这5个发现，我给出了一个'最终口径表'（就是你们现在看的这张表）。每一行都能回答'你这个结论基于什么证据、风险是什么、别人怎么可能质疑你、你怎么反驳'这四个问题。这样就算导师追问，我也有现成的说法。"
```

## 下一步立项建议（直接附加）

基于这 5 个核心结论，建议的下一步工作（优先级排序）：

1. **第 1 优先**：完成 12 篇重点文献的全文参数补全（样本量、干预频次、评估时窗、效果量、分群分析）→ 预计 1-2 周 → 产出《方法参数手册》
2. **第 2 优先**：设计一个"识别-推断"最小闭环原型（不含干预），用高教本科生的学习日志验证 → 预计 4 周 → 产出原型 + 基线对标
3. **第 3 并行**：启动伦理审查与隐私影响评估 → 预计 2 周 → 产出合规清单
4. **第 4 支撑**：补全剩余 28 篇文献的参数（按原型需要优先级调整） → 持续 → 动态更新参数库
