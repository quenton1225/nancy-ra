Overview

从敏感身份标签，转向任务过程中的学习状态与功能性支持需求

核心定位：本项目不诊断学生身份，而是探索能否通过眼动、交互和响应过程信号，理解学生在特定学习任务中的学习状态和支持需求。

术语重定位：从身份标签到支持需求

导师提醒后的关键调整 · 外部口径更稳

原有风险

问题 如果把 neurodivergent 放在项目标题或研究问题中心，容易被理解成“识别/分类学生身份”。这在学校语境和伦理语境中都比较敏感。
保留方式 neurodivergent 仍可作为内部检索词，因为 HCI、accessibility 和 learning analytics 文献仍大量使用；但不建议作为对外主语。

新表述

建议口径 SEN/SEND、students with learning difficulties、students with disabilities、functional learning needs、functional support needs。
组会句子 我们不是要判断学生“是谁”，而是要判断学生在某个学习任务中“现在可能需要什么支持”。
层面不建议强调建议强调
研究对象诊断身份或 neurodivergent 分类学习困难、可访问性障碍、支持需求
数据解释用 gaze 直接判断学生类别用过程信号辅助理解任务状态
系统目的监控或判定学生提供更及时、更匹配的学习支持

证据表

一句话解释

文献支撑 新文献支持一个更安全也更有研究价值的方向：研究目标不是学生分类，而是面向 SEN/SEND 与 functional support needs 的学习过程支持。
伦理边界 A4 尤其提醒：任何数据化工具都必须可靠、透明、非惩罚性。眼动和交互日志只能作为支持线索，不能作为惩罚或诊断依据。
方法启发 A7 给了我们方法论理由：过程数据不一定改变最终成绩，但能揭示学习调节过程差异。这正好支持 gaze + mouse + timing 的路线。

从 pilot 得到的真正结论

可行，但解释力依赖上下文字段

可以说什么

结论 1普通笔记本 webcam eye tracking 足够支撑小规模 feasibility pilot。
结论 2采集、CSV、feature extraction、scanpath 的技术链路已经跑通。
结论 3样本条件会影响数据质量，尤其是眼镜和头动。

不能说什么

边界 1不能说 gaze 已经能诊断学生类别。
边界 2不能说当前样本量足够训练模型。
边界 3不能只靠裸 gaze 坐标解释学习状态。

下一步的核心不是多收敏感数据，而是把行为上下文补足：鼠标、滚动、响应时间、AOI、校准质量、光照、眼镜、头动和研究者干预记录。

需要导师组确认的三个问题

把讨论聚焦到定位、伦理边界和下一步方法

问题 1：研究定位

是否同意把外部表述定位为 SEN/SEND / functional support needs，而不是 neurodivergent 身份识别？

问题 2：技术路线

是否同意普通笔记本 webcam eye tracking 先作为 feasibility proof，再做小规模 controlled pilot？

问题 3：数据边界

是否同意下一版只补低风险行为上下文，Zoom 录制作为独立辅助材料，而不进入主数据表？

希望得到的决策 如果上述方向成立，会后优先做 revised data collection protocol + richer-context pilot，而不是马上训练模型。
